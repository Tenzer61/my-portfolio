System Profile: eNote App

This high-fidelity system was developed by a strategic team of 3 people. In this project, I served as the primary engineer for Firebase development and database construction, ensuring a seamless and secure data architecture.

The Main Functions:

  • Real-time PDF Vault: Built a real-time database allowing users to save PDF notes securely to our system.
  • Granular Privacy: Users have full control to toggle their notes between Private or Public status for sharing.
  • In-App PDF Editor: Developed the capability for users to edit their notes directly within the app using a tactical PDF styling interface.

The Security:

  • Authenticated Entry: Access to the system is strictly limited to authorized accounts.
  • Secure Data Repository: Utilized the real-time database to store and protect user registration data within the Firebase environment.
  • Mandatory Verification: New users must complete a full verification sequence before account registration is finalized.
  • Unique Identifier Protocol: To prevent duplicate identities, each email and phone number is restricted to a single account only.
  • OTP Transmission: Integrated an OTP (One-Time Password) system that transmits security pins to users via SMS messages.

System Architecture Overview:

By utilizing an MVC pattern, the eNote app ensures that the Java controllers and Firebase backend work in perfect synchronization, creating a robust and efficient user experience.
